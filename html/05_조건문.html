조건문

    프로그래밍 코드가 해석되는 순서를 제어하는 제어문 중 한 종류
    작성된 조건식에 따라서 수행되는 코드를 변경하는 구문

    * 조건식 : 결과가 true 또는 false인 식
      ex) boolean 단독 작성, 비교 연산자, 논리 연산자

if(만약에...) 문

    
      if(조건식){
        조건식이 true인 경우 수행할 코드;
      }

입력 받은 수가 양수인지 아닌지 검사

if(만약에) - else(아니면)

    
      if(조건식) {
        조건식이 true인 경우 수행할 코드;
      }
      else {
        // else 는 조건식을 작성하는 () 가 없다
        조건식이 false 인 경우 수행할 코드;
        //유지보수의 편리를 위해 점점 사용이 줄어드는 추세
      }

0 ~ 99 사이의 난수를 발생시켜 홀 / 짝 판별하기

if - else if - else

    if      : 만약에
    else if : 앞선 조건이 아니라면(false) 다른 조건
    else    : 앞에 조건 외에(나머지, 아니면)

    
      if (조건식) {
        조건식이 true 인 경우 수행;
      }

      else if (조건식) {
        앞선 if 문 조건식이 false 이면서 
        현재 else if 문의 조건식이 true인 경우 수행;
      }

      else if (조건식) {
        // ....
      }

      else if (조건식) {
        // ....
      }

      else {
        앞선 모든 if / else if 조건이 false 인 경우 수행;
      }

중첩 if 문

    if   문 안에 if / if-else / if-else if-esle
    else 문 안에 if / if-else / if-else if-esle
      를 중첩해서 작성하는 방법

    - 여러 조건을 겹칠 때 사용함

    - 중첩은 무한히 가능함

입력 받은 수가 3의 배수가 맞는지 검사

    3 의 배수가 맞으면 : YES
    3 의 배수가 아니면 : NO 
    음수 입력          : 양수만 입력해 주세요
    입력된 값이 없다면 : 값을 입력해 주세요

Early Return

    return; : 현재실행 중인 함수를 '즉시 종료'하고 호출한 곳으로 돌아가라

    if 문 조건에 현재 수행하려는 기능의 예외 사황을 조건식으로 자성하여 
    예외 상황 발생 시 즉시 함수를 종료하게 만드는 방법

    장점 : 코드 들여쓰기가 줄어들고 간결해짐

    단점 : return 구문이 함수 내부에 분산되어 함수 코드의 전체적인 복잡도가 늘어남

두 수 A, B를 입력 받아 A 가 B 의 배수가 맞는지 확인

    ex) A : 10, B : 5 -> 배수 맞음
    
    조건 1) A 와 B 중 하나라도 입력되지 않을 경우
            -> 입력되지 않은 값이 존재합니다. 

         2) A 가 B 보다 작은 경우
            -> A 가 B 보다 작습니다.
          
         3) 1, 2 번 조건이 모두 false 이면 배수 판별 진행

switch, case, default

    switch 문
      - 식 하나의 결과로 많은 경우의 수를 처리할 때 사용하는 조건문
      -> 식의 결과와 일치하는 값을 가진 case 구문이 실행됨

    
      switch (식) { // 식의 결과가 정수 또는 문자열
        case 결과1 : 결과1의 수행 코드; break;
        case 결과2 : 결과2의 수행 코드; break;
        ...
        case 결과n : 결과n의 수행 코드; break;
        default : 모든 case가 만족하지 못하는 경우 수행
      }
